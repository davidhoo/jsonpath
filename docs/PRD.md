# 产品需求文档 (PRD)

**产品名称：** jsonpath — Go 语言 JSONPath 查询库  
**当前版本：** v2.0.2  
**文档版本：** 1.0  
**最后更新：** 2026-05-06  
**文档状态：** 草稿

---

## 1. 产品概述

### 1.1 产品定位

jsonpath 是一个 Go 语言实现的 JSONPath 查询库，提供程序化 API（Go package）和命令行工具（`jp`），用于对 JSON 数据执行 JSONPath 表达式查询。

### 1.2 产品愿景

成为 Go 生态中 RFC 9535 完全兼容的 JSONPath 标准实现，同时保留实用的非标准扩展功能。

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|----------|----------|----------|
| Go 后端开发者 | 在服务中查询 JSON 配置、API 响应 | 稳定的 API、正确的查询结果、良好的性能 |
| DevOps / SRE | 用 CLI 工具在脚本中处理 JSON 数据 | 命令行易用性、输出格式灵活 |
| 库作者 | 在上层库中集成 JSONPath 能力 | 合规性、可预测的行为、最小依赖 |

### 1.4 核心价值主张

- **合规性：** 实现 RFC 9535 标准定义的全部 JSONPath 语法和语义
- **正确性：** 查询结果与标准一致，边界行为可预测
- **兼容性：** 遵循 Go 语义版本规范，破坏性变更仅在主版本发布
- **实用性：** 保留 `keys()`、`min()`、`max()` 等实用扩展函数

## 2. 当前状态

### 2.1 已实现功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 根节点访问 `$` | ✅ | |
| 子节点访问 `.key` / `['key']` | ✅ | |
| 递归下降 `..` | ⚠️ | 不包含根节点（不符合 RFC） |
| 数组索引 `[0]`、`[-1]` | ✅ | |
| 数组切片 `[start:end:step]` | ✅ | |
| 通配符 `[*]` / `.*` | ✅ | |
| 多索引 `[1,2,3]` | ✅ | |
| 多字段名 `['name','age']` | ✅ | |
| 过滤表达式 `[?(@.price > 10)]` | ⚠️ | 缺少存在性测试、`@` 单独引用 |
| 逻辑运算 `&&`、`||`、`!` | ⚠️ | 优先级错误（左到右而非 `&&` 优先） |
| `length()` 函数 | ⚠️ | 字符串按字节计数而非字符 |
| `match()` 函数 | ⚠️ | 部分匹配而非全串匹配；使用 Go regexp 而非 I-Regexp |
| `count()` 函数 | ❌ | 签名完全错误（当前是值计数，应为节点计数） |
| `search()` 函数 | ❌ | 签名完全错误（当前是数组过滤，应为子串搜索） |
| `value()` 函数 | ❌ | 未实现 |
| 扩展函数 `keys/values/min/max/avg/sum` | ✅ | 非标准但实用 |
| CLI 工具 `jp` | ✅ | 支持文件/stdin 输入、彩色输出 |

## 4. v2.1.0 需求

### 4.1 目标

以 bug fix 名义发布行为修正和新增语法，不改变任何 API 签名，用户无需修改代码。

### 4.2 功能需求

#### FR-2.1.1 修复 `length()` 字符串计算

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | Bug fix |
| 影响文件 | `functions.go` |

**需求描述：**

`length()` 函数对字符串参数应返回 Unicode 字符数（rune count），而非字节数。

**当前行为：**
```
length("日本語") → 9  （按字节：UTF-8 编码每个汉字 3 字节）
```

**期望行为：**
```
length("日本語") → 3  （按 Unicode 字符）
```

**实现要求：**
- 使用 `utf8.RuneCountInString()` 替代 `len()`
- 仅影响字符串参数，数组/对象的 length 行为不变

**验收标准：**
- [ ] `length("日本語")` 返回 `3`
- [ ] `length("hello")` 返回 `5`
- [ ] `length("café")` 返回 `4`（含重音字符）
- [ ] `length([])` 返回 `0`（数组行为不变）
- [ ] 现有测试全部通过
- [ ] 新增 Unicode 字符串测试用例

---

#### FR-2.1.2 修复递归下降包含根节点

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | Bug fix |
| 影响文件 | `segments.go` |

**需求描述：**

递归下降操作符 `..` 应包含根节点本身的值。RFC 9535 Section 2.5.2.2 规定：`$..name` 应从根节点开始递归搜索，根节点参与匹配。

**当前行为：**
```
$..name  →  仅搜索子节点，不检查根节点自身
```

**期望行为：**
```
$..name  →  从根节点开始搜索，根节点自身也参与匹配
```

**验收标准：**
- [ ] 对于 `{"name": "root", "child": {"name": "child1"}}`，`$..name` 返回 `["root", "child1"]`
- [ ] 对于 `{"value": 1}`，`$..value` 包含根节点的 `value`
- [ ] 现有递归下降测试全部通过

---

#### FR-2.1.3 修复 `&&`/`||` 运算符优先级

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | Bug fix |
| 影响文件 | `parser.go` |

**需求描述：**

过滤表达式中 `&&` 的优先级应高于 `||`，与标准布尔代数一致。当前实现为严格的左到右求值。

**当前行为：**
```
[?(@.a == 1 || @.b == 2 && @.c == 3)]
等价于：((a == 1) || (b == 2)) && (c == 3)   ← 错误
```

**期望行为：**
```
[?(@.a == 1 || @.b == 2 && @.c == 3)]
等价于：(a == 1) || ((b == 2) && (c == 3))   ← 正确
```

**验收标准：**
- [ ] `&&` 优先级高于 `||`
- [ ] 括号可以覆盖默认优先级
- [ ] `!` 对复合表达式的求值正确（De Morgan 定律）
- [ ] RFC 9535 测试套件中相关用例通过

---

#### FR-2.1.4 修复选择器错误处理

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | Bug fix |
| 影响文件 | `segments.go` |

**需求描述：**

当选择器应用于不匹配的类型时（如对字符串使用数组索引），应返回空结果而非错误。RFC 9535 规定：结构不匹配时返回空 nodelist，不产生错误。

**当前行为：**
```
$.name[0]  →  error: cannot apply index to string
```

**期望行为：**
```
$.name[0]  →  []（空结果）
```

**规则：**
- 对字符串应用数组索引 → 空结果
- 对数字应用字段名选择器 → 空结果
- 对 null 应用任何选择器 → 空结果
- 语法错误（如未闭合的括号）→ 仍然返回 error

**验收标准：**
- [ ] 类型不匹配返回空结果而非 error
- [ ] 语法错误仍然返回 error
- [ ] `Query()` 返回的 error 仅用于真正的语法/解析错误

---

#### FR-2.1.5 添加存在性测试

| 属性 | 值 |
|------|-----|
| 优先级 | P1 |
| 类型 | 新增语法 |
| 影响文件 | `parser.go` |

**需求描述：**

支持过滤表达式中的存在性测试语法 `[?@.name]`，测试字段是否存在，不进行值比较。

**语法：**
```
[?@.name]        — 测试 @.name 是否存在（非 null）
[?@.items]       — 测试 @.items 是否存在
[?@.name.first]  — 测试嵌套字段是否存在
```

**语义：**
- 字段存在且值不为 null → 匹配
- 字段不存在或值为 null → 不匹配

**验收标准：**
- [ ] `[?@.name]` 正确过滤有 `name` 字段的元素
- [ ] `[?@.nested.field]` 支持嵌套路径
- [ ] 对 null 值的字段不匹配
- [ ] 不影响现有的比较过滤器（`[?@.name == "foo"]`）

---

#### FR-2.1.6 添加 `@` 单独引用

| 属性 | 值 |
|------|-----|
| 优先级 | P1 |
| 类型 | 新增语法 |
| 影响文件 | `parser.go` |

**需求描述：**

支持过滤表达式中 `@` 作为当前节点的独立引用，不必须跟字段名。

**语法：**
```
[?@ > 5]       — 当前节点值大于 5
[?@ == "done"] — 当前节点值等于 "done"
[?@.type == "book" && @.price < 30]  — 组合使用
```

**验收标准：**
- [ ] `[?@ > 5]` 正确过滤数值
- [ ] `[?@ == "value"]` 正确过滤字符串
- [ ] `@` 在复合表达式中正确引用当前节点
- [ ] 不影响现有 `@.field` 引用

## 5. v3.0.0 需求

### 5.1 目标

实现 RFC 9535 完全合规，引入标准类型系统，修正所有 API 签名。

### 5.2 架构需求

#### AR-3.1 引入核心类型

**需求描述：** 引入 RFC 9535 Section 2.6 定义的核心数据类型。

```go
// Node 表示 RFC 9535 节点（location + value）
type Node struct {
    Location string      // Normalized Path
    Value    interface{}
}

// NodeList 表示节点列表
type NodeList []Node

// Nothing 表示 Nothing 值（与 null 不同）
type Nothing struct{}

// LogicalType 表示三值逻辑
type LogicalType int8
const (
    LogicalNothing LogicalType = iota
    LogicalFalse
    LogicalTrue
)
```

**验收标准：**
- [ ] `Node` 类型包含 Location 和 Value
- [ ] `NodeList` 支持 JSON 序列化
- [ ] `Nothing` 与 `null` 可区分
- [ ] `LogicalType` 支持三值逻辑运算

---

#### AR-3.2 实现 Normalized Path 生成器

**需求描述：** 为每个查询结果节点生成 RFC 9535 定义的 Normalized Path。

**Normalized Path 规则：**
- 根节点：`$`
- 对象成员：`$['memberName']`（单引号包裹，特殊字符转义）
- 数组元素：`$[0]`、`$[1]`

**验收标准：**
- [ ] 每个 Node 的 Location 字段包含正确的 Normalized Path
- [ ] 特殊字符（`'`、`\`、控制字符）正确转义
- [ ] 空路径段正确处理

---

#### AR-3.3 实现 I-Regexp 解析器

**需求描述：** 实现 RFC 9485 定义的 I-Regexp 正则表达式子集，用于 `match()` 和 `search()` 函数。

**I-Regexp 特征：**
- 是 ECMAScript 正则表达式的严格子集
- 不支持反向引用、前瞻/后顾断言
- 支持 Unicode 属性 `\p{...}`
- 所有 I-Regexp 表达式都是有效的 ECMAScript 正则，但反之不然

**验收标准：**
- [ ] 解析 I-Regexp 语法
- [ ] 转换为 Go `regexp` 执行
- [ ] 拒绝非 I-Regexp 的正则表达式（如反向引用）
- [ ] 通过 RFC 9485 测试用例

### 5.3 功能需求

#### FR-3.1 变更 `Query()` 返回类型

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | 破坏性变更 |

**当前签名：**
```go
func Query(data interface{}, path string) (interface{}, error)
```

**新签名：**
```go
func Query(data interface{}, path string) (NodeList, error)
```

**变更说明：**
- 返回值从 `interface{}` 变为 `NodeList`
- 每个结果节点包含 Location（Normalized Path）和 Value
- 获取原始值需要通过 `result[i].Value`

**验收标准：**
- [ ] 返回类型为 `NodeList`
- [ ] 每个 Node 包含正确的 Location 和 Value
- [ ] 空结果返回空 `NodeList`（非 nil、非 error）

---

#### FR-3.2 修正 `count()` 函数

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | 破坏性变更 |

**当前行为：**
```
count(array, value) → float64  // 计算数组中特定值的出现次数
```

**RFC 标准行为：**
```
count(nodelist) → ValueType  // 计算节点列表中的节点数
```

**迁移方案：**
- 现有 `count(array, value)` 功能重命名为 `occurrences()`（非标准扩展）
- 新增 RFC 标准 `count(nodelist)`

**验收标准：**
- [ ] `count(@.items[*])` 返回节点数量
- [ ] `occurrences(@.items, "value")` 保留旧功能
- [ ] `count()` 接受 `NodeList` 参数

---

#### FR-3.3 修正 `search()` 函数

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | 破坏性变更 |

**当前行为：**
```
search(array, pattern) → []interface{}  // 按正则过滤数组元素
```

**RFC 标准行为：**
```
search(string, pattern) → LogicalType  // 字符串子串正则搜索
```

**迁移方案：**
- 现有 `search(array, pattern)` 功能重命名为 `filterMatch()`（非标准扩展）
- 新增 RFC 标准 `search(string, pattern)`

**验收标准：**
- [ ] `search(@.name, "pattern")` 执行子串搜索，返回 `LogicalType`
- [ ] `filterMatch(@.items, "pattern")` 保留旧功能
- [ ] 使用 I-Regexp 语法

---

#### FR-3.4 变更 `match()` 调用形式

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | 破坏性变更 |

**当前语法：**
```
?@.name.match('^S.*')     // 方法式调用，部分匹配
```

**RFC 标准语法：**
```
?match(@.name, '^S.*$')   // 函数式调用，全串匹配
```

**变更说明：**
- 调用形式从方法式（`.match()`）改为函数式（`match()`）
- 匹配行为从部分匹配改为全串匹配
- 使用 I-Regexp 替代 Go regexp

**验收标准：**
- [ ] `match(@.name, 'S')` 只匹配恰好为 `"S"` 的字符串
- [ ] `match(@.name, 'S')` 不匹配 `"AliceSmith"`（旧行为）
- [ ] 支持 I-Regexp 语法
- [ ] 旧方法式语法不再支持（解析错误）

---

#### FR-3.5 新增 `value()` 函数

| 属性 | 值 |
|------|-----|
| 优先级 | P1 |
| 类型 | 新增 |

**RFC 定义：**
```
value(nodelist) → ValueType
```

**语义：**
- 接受一个节点列表
- 如果节点列表恰好包含一个节点，返回该节点的值
- 否则返回 Nothing

**验收标准：**
- [ ] `value(@.name)` 返回单个值
- [ ] 多节点时返回 Nothing
- [ ] 空节点列表时返回 Nothing

---

evaluate(value interface{}) ([]interface{}, error)

**新签名：**

#### FR-3.7 错误处理变更

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | 破坏性变更 |

**当前行为：**
- 类型不匹配 → 返回 Go `error`
- 结构不匹配 → 返回 Go `error`

**RFC 标准行为：**
- 语法错误 → 返回 Go `error`
- 结构/类型不匹配 → 返回空 `NodeList`（非 error）

**验收标准：**
- [ ] `Query()` 返回 `error` 仅用于语法/解析错误
- [ ] 结构不匹配返回空 `NodeList`，error 为 nil
- [ ] 所有现有 error 类型保留用于真正的错误

### 5.5 扩展函数（保留但标注为非标准）

| 函数 | 签名 | 说明 | 标注 |
|------|------|------|------|
| `keys()` | `(object) → []string` | 返回对象的排序键列表 | 非标准扩展 |
| `values()` | `(object) → []interface{}` | 返回对象的排序值列表 | 非标准扩展 |
| `min()` | `(array) → number` | 数组最小值 | 非标准扩展 |
| `max()` | `(array) → number` | 数组最大值 | 非标准扩展 |
| `avg()` | `(array) → number` | 数组平均值 | 非标准扩展 |
| `sum()` | `(array) → number` | 数组求和 | 非标准扩展 |
| `occurrences()` | `(array, value) → number` | 计算值出现次数（原 `count()`） | 非标准扩展 |
| `filterMatch()` | `(array, pattern) → []interface{}` | 按正则过滤数组（原 `search()`） | 非标准扩展 |
